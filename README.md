# Lost and Found

Android-приложение для публикации и поиска объявлений о потерянных вещах в КемГУ.

Ссылка на гугл диск с лабораторными работами №3-№9
https://drive.google.com/drive/folders/1vloF5FPpnZWPoz8W99j4YNUtppAMnM3Z?usp=sharing

## Лабораторные работы

### Лабораторная работа №5 — Архитектура приложения

Проект разделён на слои `domain`, `data`, `ui` и отдельные feature-модули. Общие компоненты вынесены в модули `core`. Настроены Dependency Injection через Hilt и общая навигация между модулями.

Созданы архитектурные ветки `layer-based`, `feature-based` и `combined`. Для автоматической проверки архитектурных правил добавлены тесты с использованием библиотеки Konsist.

### Лабораторная работа №6 — Аналитика, авторизация и карта

Подключена AppMetrica для отправки аналитических событий и информации об ошибках. Добавлены события открытия экранов и успешной авторизации.

Реализована авторизация через Яндекс ID и VK ID. Токены пользователей сохраняются в защищённом хранилище `EncryptedSharedPreferences`. Добавлен раздел «О нас» с информацией об организации, картой и возможностью построить маршрут до офиса.

### Лабораторная работа №7 — Firebase

К проекту подключены Firebase Analytics, Firebase Cloud Messaging, Firebase Remote Config и Firestore.

Реализованы получение и отображение push-уведомлений, запрос разрешения на уведомления и открытие необходимого экрана после нажатия на уведомление. Remote Config используется для удалённого изменения параметров приложения. Для Firestore добавлены правила безопасности.

### Лабораторная работа №8 — Crashlytics и обработка ошибок

Подключены Firebase Crashlytics и AppMetrica Crash Reporting. Добавлена возможность создать тестовый сбой и отправить диагностическую информацию перед возникновением ошибки.

В репозиториях и ViewModel реализован отлов исключений. Некритические ошибки отправляются в Crashlytics, а пользователю отображаются понятные сообщения об ошибках.

### Лабораторная работа №9 — ИИ-помощник

В приложение добавлен ИИ-помощник на основе GigaChat. Пользователь может описать потерянную вещь и обстоятельства её пропажи, после чего получить рекомендации по поиску.

Для помощника создан отдельный feature-модуль и экран. Реализованы получение и обновление токена, обработка ошибок API и хранение ключа в `local.properties`.

## Реализация критериев семестрового проекта

### Clean Architecture

Приложение разделено на независимые слои и модули:

- `core:domain` содержит модели, интерфейсы репозиториев и use case;
- `core:data` содержит базу данных и реализации репозиториев;
- `core:services` содержит реализации внешних сервисов;
- `core:navigation` содержит общие маршруты;
- `feature:*` содержит отдельные функции приложения;
- `app` объединяет модули и запускает приложение.

Для внедрения зависимостей используется Hilt. Архитектурные ограничения проверяются тестами Konsist.

### Фоновая работа

Реализована периодическая фоновая задача `RemoteConfigSyncWorker`. Она запускается через WorkManager при наличии подключения к интернету и обновляет значения Firebase Remote Config.

Также создан `PushMessagingService`, который получает Firebase push-уведомления, отображает их пользователю и обрабатывает нажатия.

### Compose-анимации

В интерфейсе реализовано несколько видов анимаций:

- `animateContentSize()` плавно изменяет размер карточки объявления;
- `animateColorAsState()` плавно меняет цвет статуса объявления;
- нижняя панель навигации анимирует состояние выбранного раздела.

### Интеграция Compose и Android View

В разделе «О нас» карта Яндекса встроена в Compose-интерфейс через компонент `AndroidView`. На карте отображается местоположение организации. Пользователь может открыть построение маршрута до указанного адреса.

### Варианты сборки и R8

Созданы два варианта приложения:

- `demo` — версия без перехода к ИИ-помощнику;
- `full` — полная версия приложения с ИИ-помощником.

Для release-сборки включены минификация R8 и удаление неиспользуемых ресурсов.

### Качество и удобство использования

В приложении реализованы понятная навигация, обработка загрузки данных, отображение ошибок и возможность повторить неудачный запрос.

Приложение содержит основные экраны:

- лента объявлений;
- добавление объявления;
- подробная информация;
- профиль пользователя;
- раздел «О нас»;
- ИИ-помощник.
